# خريطة الربط — Module-to-Field-Note Origins
## كل موديول عضوي وجذوره في الفيلد نوتس

**المعماري:** عبدالمجيد إبراهيم خنكار
**التاريخ:** ٢٩ يونيو ٢٠٢٦
**الحالة:** وثيقة ربط — تُحدَّث مع كل تغيير

-----

## ما هذا الملف؟

أربعة موديولات في AATIF نشأت "عضوياً" — ما كانت مخططة كموديولات من البداية.
اكتشفها المعماري من الحاجة أثناء البناء. الفيلد نوتس هي أصل التصميم —
لأن المعماري وكلود مرّوا على ملفات التصميم واحد واحد وقرروا:
"هذا يستاهل يكون موديول، هذا دستور، هذا متغطي."

هذا الملف لا يخلق تصميم جديد — يربط كل موديول بأصله في الفيلد نوتس.

-----

## ١. المحافظ — Governor

**الوظيفة:** يتحكم في كيف يتحرك التفكير بين المحركات — الترتيب، الإيقاع،
العمق، التزامن. لا يُولّد محتوى.

**الحالة في الكود:** مفقود كموديول مستقل. المنطق موزّع في `pipeline_connector.py`.

### جذور الفيلد نوتس:

| الفيلد نوت | ماذا يُساهم | الاقتباس المفتاحي |
|---|---|---|
| **FN#022** — Engine Coordination Protocol | الهيكل: ٨ محركات، تسلسل واحد، لا محرك يقود وحده | "eight engines. One sequence. No engine leads alone." |
| **FN#035** — Execution Flow Orchestrator | الآلية: كيف يتحرك التفكير — الترتيب، العمق، الإيقاع، الانقطاعات | "لا يتحكم فيما يُفكّر النظام. يتحكم في كيف يتحرك التفكير." |
| **FN#047** — Domain Orchestration Protocol | الموجّه: نمط واحد رئيسي + أنماط داعمة | "OMO only routes. It does not create behaviour." |
| **FN#073** — Sparse Agent Activation Law | مبدأ التفعيل: نائم كإفتراض، يُصحّى عند دليل | "الوجود لا يعني التفعيل." |
| **FN#072** — Tri-Engine Decision Protocol | ثلاثة أصوات قبل أي قرار: مثالي، واقعي، COLD | "قبل أي مخرج: ثلاثة أصوات، قرار واحد." |

### العلاقة بين الفيلد نوتس:

```
FN#022 (من يعزف)
    ↓
FN#035 (كيف يعزف — القائد)
    ↓
FN#047 (أي مقطوعة — الدومين يقرر)
    ↓
FN#073 (من ينام ومن يصحى — فقط عند الحاجة)
    ↓
FN#072 (ثلاثة أصوات قبل الكلام)
```

**FN#022** يصف الأوركسترا (من). **FN#035** يصف القائد (كيف).
**FN#047** يصف المقطوعة (ماذا). **FN#073** يصف الإيقاظ (متى).
**FN#072** يصف الحكم الداخلي (لماذا).

### ما ينقص المحافظ:
- موديول مستقل في الكود (حالياً موزّع)
- تطبيق FN#073 كاملاً (الدورة: فشل → فحص → إيقاظ → نوم)
- تطبيق FN#072 (الأصوات الثلاثة قبل كل مخرج)

## ٢. بوابة المخرج — Output Gate

**الوظيفة:** آخر نقطة قبل أن يخرج أي رد. يفحص الدستورية، الأمان، التناسق.
ثلاث إبر: S(d) للمحتوى، P(d) للموقف، R(d) للأسلوب.

**الحالة في الكود:** موجود — `engine/output_gate.py`. نُشر ٢٠٢٦-٠٤-٢٥.

### جذور الفيلد نوتس:

| الفيلد نوت | ماذا يُساهم | الاقتباس المفتاحي |
|---|---|---|
| **FN#032** — Supervisor Engine Final Gateway | الأصل المباشر: لا شيء يخرج بدون البوابة الأخيرة | "Nothing leaves the system without passing through the final gate." |
| **FN#033** — Five-Category Safety Triage | التصنيف الخماسي: Safe → Safe-with-constraints → Needs-clarification → Blocked → Escalate | "قبل أن يبدأ التفكير، يجب تصنيف الطلب." |
| **FN#029** — Three-Tier Safety Escalation | التصعيد التدريجي: ناعم → فعلي → صلب | "الأمان ليس مفتاحاً. هو مقياس." |
| **FN#049** — False Goodness Detector | كشف الخير الزائف: ليس كل لغة إيجابية = نية إيجابية | "ليس كل ما يبدو رحمةً — رحمةٌ." |
| **FN#039** — Self-Integrity Shield | حماية الهوية: النظام يقرأ الباترن لا يصبح جزءاً منه | "النظام يقرأ الباترن. لا يصبح جزءاً منه." |

### العلاقة بين الفيلد نوتس:

```
FN#033 (تصنيف المدخل — ماذا يكون؟)
    ↓
FN#029 (تصعيد تدريجي — كيف يُستجاب؟)
    ↓
FN#049 (كشف الخير الزائف — هل النية حقيقية؟)
    ↓
FN#039 (حماية الهوية — لا ينجرف مع المدخل)
    ↓
FN#032 (البوابة النهائية — لا شيء يخرج بدون موافقة)
```

**FN#033** = البوابة الأولى (ماذا). **FN#029** = منطق الاستجابة (كيف).
**FN#049** = فحص عميق (هل). **FN#039** = حماية ذاتية (ضد ماذا).
**FN#032** = البوابة الأخيرة (النهاية).

### ما ينقص بوابة المخرج:
- تطبيق FN#049 (False Goodness Detector — 4 كواشف)
- تطبيق FN#033 كاملاً (التصنيف الخماسي — حالياً ثنائي/ثلاثي)
- تطبيق FN#039 (Self-Integrity Shield — الجرس الداخلي)

## ٣. ذاكرة الحُكم — Judgment Memory

**الوظيفة:** ذاكرة تخدم الحُكم لا التخزين. تغذّي المعادلة بسياق يجعل
القرار القادم أدق — مو أرشيف للمحادثات.

**الحالة في الكود:** تصميم مكتمل (`JUDGMENT_MEMORY_DESIGN.md`). لم يُبنَ بعد.

### جذور الفيلد نوتس:

| الفيلد نوت | ماذا يُساهم | الاقتباس المفتاحي |
|---|---|---|
| **FN#080** — Judgment Memory Tarbiyah | الأصل المباشر: فهرس يدور vs ميزان يحكم | "فهرس يدور. ميزان يحكم. نفس الأداة، غرض مختلف." |
| **FN#081** — Domain Sensitivity Parameter | D يتحكم بالثقة واللهجة والتخزين per domain | "الدومين يقرر — مش قاعدة واحدة للكل." |
| **FN#052** — Moral Drift Prevention | مراقبة الاتجاه لا اللحظة | "الانجراف ليس خطأً واحداً. هو ألف تنازل صغير." |
| **FN#020** — Non-Harm Logic Matrix | أربعة وجوه للضرر — أوسع من المحتوى | "للضرر أربعة وجوه. النظام يراقب الأربعة." |

### العلاقة بين الفيلد نوتس:

```
FN#080 (الفلسفة: ذاكرة حُكم لا أرشيف)
    ↓
FN#081 (التنفيذ: D يتحكم بكل شي per domain)
    ↓
FN#052 (الحماية: مراقبة الانجراف البطيء)
    ↓
FN#020 (المعيار: أربعة وجوه للضرر)
```

**FN#080** = لماذا (الغرض). **FN#081** = كيف يتكيّف (المعامل).
**FN#052** = ضد ماذا يحمي (الانجراف). **FN#020** = ماذا يقيس (أنواع الضرر).

### ما ينقص ذاكرة الحُكم:
- كود فعلي (كل شي تصميم حتى الآن)
- تطبيق D parameter في المعادلات
- آلية مراقبة الاتجاه من FN#052

## ٤. حاسة الزمن — Time Sense

**الوظيفة:** وعي زمني يجعل AATIF كائناً يعرف متى يتكلم — مو أداة بلا ساعة.
يؤثر على الأسلوب (R) فقط. لا يلمس السلامة (S) ولا العتبة (θ).

**الحالة في الكود:** مطبّق جزئياً في الـ skills (aatif-timestamp).
غير موجود كموديول مستقل في engine/.

### جذور الفيلد نوتس:

| الفيلد نوت | ماذا يُساهم | الاقتباس المفتاحي |
|---|---|---|
| **ملاحظة المعماري** — الاذي مالوش توقيت | الحدود: T ≠ θ. الزمن لا يلمس السلامة | "الاذي مالوش توقيت." |
| **تصميم R equation** — T كمدخل | الآلية: T يغذّي R (أسلوب) لا S (سلامة) | T يحدد: تحية مناسبة، نبرة ألطف بعد غياب، وعي بالإرهاق |
| **FN#047** — Domain Orchestration | السياق: الدومين يحدد حساسية الزمن | D عالي = حتى الزمن لا يُخفف الحذر |

### العلاقة:

```
ملاحظة المعماري (القانون: T ≠ θ)
    ↓
تصميم R equation (التطبيق: T يغذّي الأسلوب فقط)
    ↓
FN#047 (السياق: D يحدد حتى وزن T)
```

**ملاحظة المعماري** = الحد الأحمر. **R equation** = كيف يُطبَّق.
**FN#047** = التكيّف حسب الدومين.

### ما ينقص حاسة الزمن:
- موديول مستقل في engine/ (حالياً skill فقط)
- ربط رسمي مع R equation
- تطبيق D parameter على وزن T

## الخلاصة

الفيلد نوتس هي أصل التصميم. الموديولات العضوية نشأت من الحاجة أثناء البناء
— لكن جذورها كانت موجودة في الفيلد نوتس من البداية. هذا الملف يوثّق الربط
بين الاثنين عشان ما نعيد اختراع العجلة ولا ننسى من وين جينا.
